# RentRights — 설계 스펙 (v1, City of Los Angeles)

- **작성일:** 2026-06-02
- **상태:** 설계안 (사용자 검토 대기)
- **한 줄 정의:** LA 세입자가 *주소만 입력하면* 자기 집에 적용되는 임대법·합법 인상 한도·정당사유 퇴거 권리를 **추정·안내**하는 무료 오픈소스 도구
- **선정 경위:** "LA 커뮤니티 임팩트" 목표로 11개 후보 × 12 페르소나(전문가 6 + 일반인 6) 평가에서 종합 1위 (49/60). 최대 도달층(시 인구의 65%가 세입자), 최고 빌드 용이성, 파트너 없이도 가치 제공 가능.

---

## 1. 문제와 목표

- LA 세입자 약 400만 명(시 인구의 65%)은 **자기 집에 어떤 임대법이 적용되는지, 합법 인상 한도가 얼마인지** 모른다. 정보는 4개 이상 기관 사이트에 흩어져 있고, 공식 도구(ZIMAS 등)는 건물 단위·전문가용이라 일반인이 못 쓴다.
- 집주인은 세입자가 한도를 모른다는 이유로 위법 인상을 시도한다.
- **목표:** 주소 한 줄로 "내 권리"를 평이한 언어로, 모바일·다국어로, 무료·개인정보 비저장으로 제공. 비영리·오픈소스·1인 개발 가능.

## 2. 범위 (v1)

**포함**
- 주소 → 임대 레짐 **추정**(RSO / AB 1482 / 시티와이드 JCO만 해당) + **신뢰도**
- 합법 **연 인상 한도**($/%)
- **정당사유 퇴거·통지기간·이주비** 권리 (읽기 전용 안내)
- "LAHD에 확인" + "무료 법률지원 찾기" 연결 (결과 **문자(SMS)로 받기**는 v1/v2 여부 §15에서 확정)
- 지리 범위: **City of Los Angeles 한정** (RSO + AB 1482 + 시티와이드 JCO)

**제외 (v1)**
- LA County RSTPO 및 기타 시(Santa Monica·WeHo 등) — v2+
- 민원 접수/법률 서신 생성/퇴거 소송 대응 (별도 도구 "EvictionShield" 영역)
- 사용자 PII 저장
- 권위 있는 등록부 직접 조회 (불가능 — §3 참조)

## 3. 핵심 설계 원칙 — "정직한 추정기"

검증 결과(2026-06-02): **LA시 RSO 등록부는 공개 API가 없다.** (`data.lacity.org`에 데이터셋 없음 / LAHD Rent Registry는 로그인 전용·2026-05 폐지 / ZIMAS·DCBA는 API 없는 SPA.)

→ RentRights는 "조회기"가 아니라 **공개 Assessor 기록 + 법 규칙 기반 추정기**다. RSO의 *법적 정의*("1978-10-01 이전 준공 + 2세대 이상 + LA시")가 곧 규칙이므로 대부분 정확히 분류 가능하나, 본질적으로 추정이다.

**따라서 모든 결과는** ① 판단 **근거**(준공연도·세대수·관할)를 먼저 보여주고 ② **신뢰도**를 표기하며 ③ "LAHD에서 확인" 경로를 항상 제공한다. (결과 표시 = **근거 우선형, 옵션 C**.) 단정 금지. 이는 데이터 현실이 강제하는 필수 사항이며, 평가 패널의 변호사·데이터 엔지니어 경고와 일치한다.

## 4. 사용자 흐름 (모바일 우선, 4 화면)

1. **주소 입력** — 무료·오픈소스·개인정보 비저장 전면 표기, 언어 토글(EN/ES)
2. **확인 질문** *(신뢰도 낮을 때만 노출)* — 공개데이터로 판별 불가한 1~3개 질문(§5.3)
3. **결과 (핵심)** — 근거 → "아마도 RSO/AB1482/JCO" + 신뢰도 + 합법 인상 한도 + "확정 아님, LAHD 확인" 배너
4. **권리 상세 + 도움받기** — 정당사유·통지·이주비(읽기 전용) → 문자 받기 / LAHD 문의 / 무료 법률지원 / "마지막 검증일·법률 검토" 표기

## 5. 레짐 판별 엔진 (핵심)

### 5.1 파이프라인 (전부 검증된 공개 API, 무료·무인증)
1. **관할 판별** — Census Geocoder(geographies, `layers=27` Incorporated Places) → "Los Angeles city" 여부
2. **주소 → AIN** — LA County Assessor **PAIS MapServer** (`SANUM`+`SASTR` 매칭; 퍼지 매칭·ZIP 폴백 필요)
3. **AIN → 필지 사실** — Assessor **Rolls FeatureServer** (`YearBuilt`, `Units`, `UseCode`; 최신 `RollYear` 필터) → 보조로 `parceldetail` API(용도 설명·zoning; `NumOfUnits`는 신뢰 불가)
4. **규칙 엔진**(순수 함수) → 레짐 + 신뢰도 + (필요시) 확인 질문 목록

> 라이브 검증 예: `1411 Murray Dr` → LA city / AIN 5425003009 / 1931년·6세대·UseCode 0500 → **RSO 추정 = True** (실제 200 응답 확인)

### 5.2 규칙 (검증된 법적 정의 인코딩)
- LA시 + 준공 **≤ 1978-10-01** + **2세대 이상** + 주거 용도 → **RSO** (신뢰도 높음)
- LA시 + 준공 ≤ 1978 이나 모호(콘도/단독+ADU/세대=1/준공=1978년) → **확인 질문** 후 결정
- LA시 + 준공 > 1978 + (다세대 또는 준공 15년 경과) → **AB 1482** (면제 가능 → 확인 질문)
- LA시 + 단독주택/콘도 또는 AB1482 면제 → **JCO만 해당** (시티와이드 정당사유는 여전히 보호)
- LA시 외 → 범위 외 안내 + 해당 관할 포인터(WeHo·Santa Monica·비편입 County)

### 5.3 신뢰도 모델 & "확인 질문" 트리거 (공개데이터로 판별 불가 — V1 검증)
| # | 트리거 | 확인 질문 |
|---|--------|-----------|
| 1 | UseCode가 콘도/아파트 구분 불가 | "콘도(분양) 건물인가요?" |
| 2 | 소유주 면제신청 여부(공개 API 없음) | "집주인이 LAHD에 면제/예외 신청을 했나요?" |
| 3 | 세대수=2 (단독+ADU 가능) | "다른 한 채가 별도 주택(ADU/별채)인가요?" |
| 4 | 준공연도=1978 (10/1 기준 모호) | "준공 허가(CO)가 1978년 10월 이전인가요?" |
| 5 | 신축(준공 15년 내) AB1482 면제 | "집주인이 AB 1482 면제 통지를 줬나요?" |
| 6 | 한도 적용에 현재 임대료 필요 | "현재 월세와 마지막 인상 시점은?" |
| 7 | Census가 편입지 미반환 | "비편입 LA County 지역인가요?" |

- **신뢰도 높음** = 데이터 명확·모호 트리거 없음 / **중간** = 경미한 모호(보조질문 1개) / **낮음** = 핵심 모호(질문 필요, 단정 금지·가능성 범위로 표시)

## 8. 아키텍처 & 스택

- **Next.js (App Router) + TypeScript**, 모바일 우선. 외부 공개 API 호출은 **서버 라우트**로 프록시(CORS 회피 + 캐싱 + 키 불필요 유지).
- **DB 불필요(v1)** — 무상태 요청. 필지 결과는 **캐시**(Assessor 데이터는 연 단위 변경) → API 호출/지연 절감(경량 KV 또는 엣지 캐시).
- **i18n**: 출시 시 **영어 + 스페인어**, 구조는 한국어/아르메니아어/중국어 확장 가능.
- **개인정보 비저장**: 결과는 요청 시 계산. SMS는 전화번호를 *일시적으로만* 사용하고 영구 저장하지 않음(또는 SMS는 v2로).
- **배포**: 셀프호스트(기존 인프라) 또는 Vercel — *미해결(§15)*.
- **staleness 가드**: `legal-constants.json`의 날짜 기반으로 현재 적용값 자동 선택 + 갱신 지연 경고.

## 9. 컴포넌트(모듈) — 각자 단일 책임·독립 테스트 가능
1. `jurisdiction` (Census) — 주소 → 관할
2. `parcel` (PAIS + Rolls) — 주소 → 필지 사실(퍼지 매칭·폴백)
3. `rules-engine` (순수 함수) — 필지 사실 + 사용자 답변 → {레짐, 신뢰도, 필요한 질문[]}
4. `legal-content` — 상수 + 권리 문구(레짐별·i18n)
5. `confirming-questions` — 트리거→질문 제어
6. `result-presenter` — 근거 우선(C) 렌더
7. `sms`(선택) / `get-help`(LAHD·Stay Housed LA·SAJE 디렉터리)

## 10. 오류 처리 & 엣지케이스
- 주소 미발견/모호 → 후보 제시·재입력 유도
- 외부 API 장애(Census/Assessor) → **질문 우선 경로로 폴백**(데이터 없이 사용자 답변으로 추정), 무음 실패 금지
- 레짐 모호/신뢰도 낮음 → 단정 금지, 가능성·다음 행동 안내
- 관할 외 → 명확한 리다이렉트(해당 시/카운티 안내)
- 상수 갱신 지연 → staleness 배너

## 11. 신뢰·법적·안전
- "추정이며 법률 조언 아님 · LAHD에서 확인" 고지 상시
- 출시 전 **법률단체(SAJE/LAFLA/Stay Housed LA 등) 1회 검토** — 검토자 표기로 커뮤니티 배포 신뢰 확보(일반인 페르소나의 "지인/단체 보증" 요건)
- **오픈소스** 공개 → 단체가 감수·기여 가능
- 접근성: 모바일 우선·저문해 평이언어·스크린리더

## 12. 테스트 전략
- **단위:** 규칙 엔진 — 알려진 주소/필지 골든셋 → 기대 {레짐, 신뢰도} (검증된 `1411 Murray Dr`=RSO 포함)
- **경계값:** 1978 컷오프, 세대수 1/2, COO 날짜 모호
- **계약/통합:** Census·Assessor 실 API 계약 테스트 + 고정 픽스처 + **스키마 드리프트 카나리 경보**(데이터 엔지니어 페르소나 우려)
- **법률 상수:** 특정 날짜 → 올바른 적용값 선택 로직 검증
- **E2E:** 4화면 흐름

## 13. 배포 & 성공지표
- **파트너 준비형** 설계 — 세입자 단체 공동브랜딩/보증 가능(신뢰 보증 경로)
- 지표: 조회수, 확신 답변 비율, SMS/도움받기 클릭, 단체 채택 수
- 정직한 비목표: "퇴거 방지 수"는 통제 밖 → 성과 지표로 쓰지 않음

## 14. 범위 외 / 향후 (v2+)
